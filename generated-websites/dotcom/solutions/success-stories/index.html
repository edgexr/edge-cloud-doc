    MNO Success Stories

    MobiledgeX works with MNOs, Edge Native SW developers and standards bodies to drive the success of MNOs in the Telco Edge Cloud market. Here are some of our TEC success stories:

    5G QoS API for Edge Native Developers

    Problem

    Leading European MNO had to setup 5G QoS policies on behalf of their edge native app partners.They wanted to enable mobile app developers to be self sufficient in provisioning their desired 5G QoS policies

    Solution

    MobiledgeX worked with the MNO and Linux Foundation to define  an easy to use, standards based 5G QoS API. Once defined MobiledgeX implemented the API and necessary southbound 5G RAN integrations. The MNO conducted a successful QoS API trial with an edge native app developer and the Linux Foundation is working to standardize it.

    MNO Edge Cloud Federation

    Problem

    Developers of Edge Native Solutions want a consistent QoE for users even when they’ve roamed off of their home network’s edge cloud and a single bill for edge services consumed.

    Solution

    MobiledgeX collaborated with GSMA TEC, an edge computing focused team of global MNOs and technologists, to define a standard for a common edge native app developer facing API and inter-MNO edge cloud roaming service. MobiledgeX implemented the standards and supported a trial demonstrating the federation and roaming of an edge native Computer Vision application “backend” between the  Telco Edge Clouds of two participating MNOs while maintaining consistently high user experience. The result was a technically validated path to an industry standard edge computing Operator Platform/Hub.

    Telco Edge enhanced AR mobile application user experience

    Problem

    Graphics quality for AR mobile application (i.e. virtual human tour guide) was limited by “device side” rendering capabilities. MNOs and their AR app partners sought a telco edge solution that would improve user experience while keeping latency low.

    Solution

    Remote AR graphics rendering service and VPS was deployed on multi-operator MobiledgeX platform to get past device based rendering limitations while keeping latency as low as possible.

    Vehicle Collision Warning System

    Problem

    Automotive/autonomous mobility ISV required reliable and predictable low latency to ensure safe and efficient operations  of their “collision warning system” on MEC infrastructure. They explored how to build a future-proof set up to support low latency requirements for smart mobility/telematics services.

    Solution

    MobiledgeX worked with two European operators to expose their varied edge clouds as a unified edge cloud services development and deployment layer. The ISV was able to validate that their collision warning system application backend could perform consistently even when roaming between the operators’ edge clouds.

    Edge Enhanced Holographic Conferencing

    Problem

    “Metaverse” type holographic conferencing software developers were challenged by mobile device limitations and poor latency when using cloud resources external to the telco network. Furthermore MNOs hosting this holographic conferencing application were experiencing high transport costs

    Solution

    European MNOs interconnected through MobiledgeX to provide seamless edge discovery, low latency and reduction in jitter for enhanced metaverse experience. By working together, MNOs were able to save on transport bandwidth and offloading graphic processing from the device to the network.

    Edge-Enabled Autonomous Search and Rescue Using Drones

    Problem

    Developer of drone/robot swarm management for search and rescue and other use cases needed the ability to launch a universal drone management app across multiple telco edge cloud environments serviced by multiple MNOs. Drone swarm management for mission critical public safety applications demands a very low latency that is difficult to achieve using cloud resources external to the mobile network

    Solution

    MobiledgeX was leveraged by two MNOs  to provide consistently high  QoS for the drone swarm management application regardless of which MNO’s cloudlet was serving the “app backend”
